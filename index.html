HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • An ordered list displays with ordered numbers. An unordered list displays with bullets
  • Each heading is listed in order as "h1" through "h6" with corresponding text size
  • An "id" refers to a unique element, while a "class" can appear more than once
  • Meta elements communicate with the browser, but is not seen by the user
  • A "Span" can refer to line-level text within a block. A "Div" groups larger blocks together
  • A "link" element connects to an external file. "rel" describes the relationship and "href" is the location (URL)

CSS

  • Each selector has a Declaration which is a "Property: Value" like "color: blue"
  • Margin indicates the amount of space OUTSIDE or AROUND an element
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside the element
  • The "*" star selector is wild card and applies something to ALL elements on the page
  • You can apply declarations to multiple selectors by adding both, separated by a comma
  • For an image selector "img" the "display: block" declaration will make each image start on a new line

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git commit: commits your changes
  • git clone + link: creates a local copy of remote repository
  • git push origin + branch name: pushes a branch to the repository
  • git pull origin + branch name: pulls changes from remote repository
  • git checkout + branch name: switched to that branch

JavaScript

  • A variable is a named container that allows us to store data in our code
  • Must first declare a variable using "let" variable = x. Then can change later saying variable = x
  • Variables can be Strings (text sequence in quotes), Numbers (no quotes), Boolean (true/false) arrays (in brackets separated by commas) or Objects (anything)
  • Can write comments using "/* comment */" "// comment" which will be ignored by browser but there for your reference
  • A (+) adds 2 numbers or concatenates strings. (-) for subtraction. (*) for multiplication. (/) for division
  • An "=" assigns a variable. "===" is strict equality and tests if two items are the same and same data type returned as true/false
  • To denote not use (!). To note does not equal use (!==)
  • Conditionals are "if...else" statements
  • The parenthases following a function contains arguments, separated by commas if muliple. These are inputs.
  • Events are activity from the user that a website listens for in order to employ some sort of action (addEventListener("event", () {result}))
  • Adding "const" prior to a variable means it cannot be reassigned and remains constant value. Adding "let" means it can be changed later
  • Control flow is the order in which a computer executes code in a script. In JS reads top to bottom and left to right
  • Control flow can be interrupted by conditional statements (if, else if, else) or by loops (for or while)
  • An array stores a group of related data within brackets. Each variable counts up item [0], item [1], item [2], etc
  • First define an array: const var = [item1, item2, item3, etc]. To iterate over a loop: for(let x = 0; x < var.length; x++)
